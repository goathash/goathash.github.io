GoatHash

What is GoatHash?

GoatHash is a fully client-side, zero-server key–value hash lookup system. Simply paste an MD5 or NTLM hash, and the site instantly returns the original plaintext— all without any backend processing.

How does it work?

The application uses a pre-computed dictionary of hashes stored in thousands of small text files (“chunks”). When you search for a hash, the JavaScript logic:

  1. Normalizes the input to lowercase.
  2. Computes a fast FNV-1a 32-bit hash.
  3. Uses the hash modulo the total number of chunks (2860) to determine the correct file.
  4. Fetches that .txt chunk via HTTP.
  5. Scans it line by line for an exact match and displays the corresponding value.

Everything runs entirely inside your browser—no server, no database, and virtually no latency, aside from fetching one small text file.

How is this possible without a database?

We generated the full hash-to-plaintext mapping offline using Python, then split it into 2,860 lightweight chunks. These, along with meta.json, are shipped with the static website. The browser loads only the single relevant chunk at search time, keeping performance and memory usage efficient.
